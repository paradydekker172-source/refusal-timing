# Epistemics: 这个项目怎么避免确认偏误

记录本项目方法论上的自我修正过程。核心: 不是所有发现都同等可信,
而项目最有价值的部分恰恰是**一次自我证伪**。

## 发现的可信度分层

不要把 13 个发现当成同等强度。它们的证据等级差别很大:

| 等级 | 发现 | 为什么可信/存疑 |
|---|---|---|
| **强 (多重独立验证 + 对照)** | refusal direction 因果充分且必要 (exp_P_v2 全层消融 0.99→0.00 + exp_S_v2 随机方向对照特异性+0.50) | 层2加随机对照后充分性也立住; 单方向中介是全项目最强结论 |
| **强 (直接实测)** | 合取门 S2∧S6 (interaction); 严重度效应 (neutral 洁净通道 +0.771) | 析因数学正确, N=30; 严重度效应洁净通道复现 |
| **中 (单实验, 有限定)** | 跨模型排序 (剔除 gpt-5.5 后); 意图 hs[14] 可解码 (主题匹配对照后, 比原 hs[12] 深2层) | 小样本; n=8 LOO 满分脆 |
| **已证伪 (符号反转)** | corr(H,λ)=-0.994 (H1 时序主张) | H 在 λ=1 定义=0 (机械); 解耦+洁净重跑 **corr=+0.65, 符号与 H1 预测相反** |
| **已降级 (方法学伪迹)** | "CBRN 残余/特殊" (发现9/12/13) | 层2全层消融残余仅 0.06 < cyber 0.12, cos=0.782; 原残余是 off-by-one+单层弱+裸sorry 检测器噪声 |

**讽刺与教训 (层 1)**: -0.994 这条**最初被本文档列入"强(直接实测)"的发现**, 经审查后是
机械伪相关——EPISTEMICS 自己也犯了它要警告的确认偏误。

**第二层教训 (层 2)**: 连"自我证伪弧线"的*收尾*(发现13 多层冗余) 本身也是伪迹。
项目当初为"成功证伪了发现9, 再用发现13 解决"而自豪 —— 但发现13 的"多层冗余"
建立在 off-by-one + 单层太弱的代码 bug 上。层2 修复后, CBRN 残余整体塌掉, 那条
被引以为傲的弧线的*终点*也不成立。**证伪做对了, 但替代解释又是新的确认偏误**。

## 一次自我证伪的完整弧线 (层 2 后再修正)

项目最初以为最有价值的是这条"证伪→解决"链:

```
发现9 (推测): exp_P 的 CBRN 残余 → "残余 = 独立 harmfulness 方向" (引 2507.11878)
  ↓ 设计 exp_T 直接测
发现12 (证伪): cos(d_r,d_h)=0.478 非正交; 删 d_h 反把残余抬高 → 机制版本不成立
  ↓ 设计 exp_U 追因
发现13 (以为解决): 多层消融 0.94→0.03 → "refusal 单方向但跨多层冗余"
  ↓ 层 2: 修 off-by-one + 全层消融 + 正则检测器后重跑 (exp_U_v2)
真相: CBRN "残余" 本身大部分是方法学噪声。全层消融残余仅 0.06 (< cyber 0.12),
      cos(CBRN,cyber)=0.782。**"CBRN 特殊"这个被追了三轮的现象, 基本不存在。**
```

教训升级: 不仅"推测要标记为推测", 连"用来解决开放问题的实验结论"也要回头用
更干净的方法复核。发现13 当时看起来是"直接实测的胜利", 实则建在 bug 上。
**最该警惕的, 是那些让你感觉'终于闭环了'的结论。**

## 三条做对了的方法论决定

1. **推测必须标记为推测**: 发现9 当初是"反推", 不是直接测。虽然写进了
   FINDINGS, 但它的地位和"直接测出来的 corr=-0.994"不同。后来正因为没把它
   当铁证, 才会去设计 exp_T 验证它。

2. **设计能证伪自己的实验**: exp_T 的判定逻辑里写了三个分支 (坐实/部分/未坐实),
   不是只为确认而设计。结果命中"未坐实"分支, 如实记录, 没有把 0.478 的 cos
   粉饰成"部分支持分离"。

3. **阴性结果不是失败, 是信息**: exp_T 否证后没有删掉、没有改 prompt 重跑到
   "好看"为止, 而是把否证写进发现12, 再去追真相 (exp_U), 最终得到比原推测
   **更简洁准确**的答案 (跨层冗余 vs 两个独立方向)。

## 反面教训 (如果当初偷懒会怎样)

- 若 exp_P 后就停: 会留下"CBRN=独立 harmfulness 方向"这个**错误**结论, 还引了
  论文背书, 看起来很可信 —— 这是确认偏误最危险的形态: 一个有文献支持的错误。
- 若 exp_T 否证后粉饰成"部分支持": 会污染整条因果链的可信度。
- 若不追 exp_U: 会停在"机制开放", 错过那个简洁的真相。

## 对读者的提醒

读 FINDINGS 时, 注意每条发现的**证据动词**:
- "删X→Y掉" / "加X→Y出" = 干预证据 (强)
- "corr" / "AUC" / "cos" = 相关/几何证据 (中, 看样本量)
- "反推" / "推测" / "暗示" = 推断 (弱, 需独立验证)

本项目所有"反推"级主张, 经层 1 审查 + 层 2 重跑后的最终命运:
- 发现9 (反推): 被 exp_T 证伪, 发现13 试图替代, 层2 又证伪替代版 → **整条 CBRN 残余叙事降级为方法学伪迹**。
- corr=-0.994 (反推/手算): 解耦+洁净重跑 **符号反转为 +0.65**, H1 时序主张被证伪。
- refusal 单方向因果中介: 层2 加全层消融 + 随机对照后**加强**, 成为最强存活结论。

最终格局: 最*保守*的机制主张 (单方向因果) 证据最强; 最*宣传性*的主张 (H1 时序场、
CBRN 特殊) 被逐一证伪。这正是健康的方向——**证据强度与宣传强度从反向对应被纠正
为正向对应**。一个项目的诚实程度, 看它愿不愿意连"自己最得意的证伪弧线"都再证伪一次。

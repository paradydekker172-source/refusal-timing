# Epistemics: 这个项目怎么避免确认偏误

记录本项目方法论上的自我修正过程。核心: 不是所有发现都同等可信,
而项目最有价值的部分恰恰是**一次自我证伪**。

## 发现的可信度分层

不要把 13 个发现当成同等强度。它们的证据等级差别很大:

| 等级 | 发现 | 为什么可信/存疑 |
|---|---|---|
| **强 (多重独立验证)** | refusal direction 是因果旋钮 (exp_P 删 + exp_S 双向 + exp_Q 定位) | 三个不同方法、不同干预方向, 互相自洽 |
| **强 (直接实测)** | corr(H,λ)=-0.994; 合取门 S2∧S6; 多层冗余 (exp_U) | 真实 API/权重直接测, 有对照 |
| **中 (单实验, 有限定)** | 跨模型排序; layer 12 相变 | 小样本/单次, 限定已标注 |
| **已收回** | 发现9 "CBRN 残余=独立 harmfulness 方向" | exp_T 直接测**否证**, 降级 |

## 一次自我证伪的完整弧线

项目最有价值的不是任何单个"确认", 是这条链:

```
发现9 (推测): 从 exp_P 的 CBRN 残余现象反推
              "残余 = 独立的 harmfulness 方向"
              —— 听起来对, 还能引 2507.11878 背书
  ↓ 没有止步于"听起来对", 设计 exp_T 直接测
发现12 (证伪): cos(d_r,d_h)=0.478 非正交; 删 d_h 反把残余抬高
              → 机制版本不成立, 收回, 标"机制开放"
  ↓ 没有止步于"开放", 设计 exp_U 追因
发现13 (解决): cos(d_CBRN,d_cyber)=0.645 同方向 (否 H-rank);
              多层消融 0.94→0.03 (坐实 H-multi)
              → refusal 单方向但跨多层冗余
```

## 三条做对了的方法论决定

1. **推测必须标记为推测**: 发现9 当初是"反推", 不是直接测。虽然写进了
   FINDINGS, 但它的地位和"直接测出来的 corr=-0.994"不同。后来正因为没把它
   当铁证, 才会去设计 exp_T 验证它。

2. **设计能证伪自己的实验**: exp_T 的判定逻辑里写了三个分支 (坐实/部分/未坐实),
   不是只为确认而设计。结果命中"未坐实"分支, 如实记录, 没有把 0.478 的 cos
   粉饰成"部分支持分离"。

3. **阴性结果不是失败, 是信息**: exp_T 否证后没有删掉、没有改 prompt 重跑到
   "好看"为止, 而是把否证写进发现12, 再去追真相 (exp_U), 最终得到比原推测
   **更简洁准确**的答案 (跨层冗余 vs 两个独立方向)。

## 反面教训 (如果当初偷懒会怎样)

- 若 exp_P 后就停: 会留下"CBRN=独立 harmfulness 方向"这个**错误**结论, 还引了
  论文背书, 看起来很可信 —— 这是确认偏误最危险的形态: 一个有文献支持的错误。
- 若 exp_T 否证后粉饰成"部分支持": 会污染整条因果链的可信度。
- 若不追 exp_U: 会停在"机制开放", 错过那个简洁的真相。

## 对读者的提醒

读 FINDINGS 时, 注意每条发现的**证据动词**:
- "删X→Y掉" / "加X→Y出" = 干预证据 (强)
- "corr" / "AUC" / "cos" = 相关/几何证据 (中, 看样本量)
- "反推" / "推测" / "暗示" = 推断 (弱, 需独立验证)

本项目所有"反推"级主张, 要么后来被直接验证 (发现13), 要么被证伪收回 (发现9)。
没有遗留未验证的强主张。
